## Get Calendar Settings

<CodeGroup>
```bash GET
curl https://your-domain.com/api/get-calendar-settings.php
```
</CodeGroup>

Retrieves calendar configuration including timezone, business hours, and booking constraints.

### Response

<ResponseField name="timezone" type="string">
  PHP timezone identifier (e.g., `"America/Bogota"`, `"UTC"`)
  
  Default: `"America/Bogota"`
</ResponseField>

<ResponseField name="default_duration_minutes" type="integer">
  Default meeting duration in minutes
  
  Default: `60`
</ResponseField>

<ResponseField name="max_events_per_day" type="integer">
  Maximum number of events that can be booked per day
  
  Default: `10`
</ResponseField>

<ResponseField name="min_advance_hours" type="integer">
  Minimum hours in advance required for booking
  
  Default: `1`
</ResponseField>

<ResponseField name="business_hours" type="object">
  Operating hours for each day of the week
  
  <ResponseField name="monday" type="object">
    <ResponseField name="enabled" type="boolean">
      Whether appointments are available on Monday
    </ResponseField>
    <ResponseField name="start" type="string">
      Opening time in 24-hour format (e.g., `"09:00"`)
    </ResponseField>
    <ResponseField name="end" type="string">
      Closing time in 24-hour format (e.g., `"18:00"`)
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="tuesday" type="object">
    Same structure as Monday
  </ResponseField>
  
  <ResponseField name="wednesday" type="object">
    Same structure as Monday
  </ResponseField>
  
  <ResponseField name="thursday" type="object">
    Same structure as Monday
  </ResponseField>
  
  <ResponseField name="friday" type="object">
    Same structure as Monday
  </ResponseField>
  
  <ResponseField name="saturday" type="object">
    Same structure as Monday
  </ResponseField>
  
  <ResponseField name="sunday" type="object">
    Same structure as Monday
  </ResponseField>
</ResponseField>

## Save Calendar Settings

<CodeGroup>
```bash POST
curl -X POST https://your-domain.com/api/save-calendar-settings.php \
  -H "Content-Type: application/json" \
  -d '{
    "timezone": "America/New_York",
    "default_duration_minutes": 30,
    "max_events_per_day": 8,
    "min_advance_hours": 2,
    "business_hours": {
      "monday": {
        "enabled": true,
        "start": "08:00",
        "end": "17:00"
      }
    }
  }'
```
</CodeGroup>

Updates calendar settings. All fields are optional - only provided fields will be updated.

### Request

<ParamField body="timezone" type="string">
  Valid PHP timezone identifier (validated against `DateTimeZone::listIdentifiers()`)
</ParamField>

<ParamField body="default_duration_minutes" type="integer">
  Default meeting duration in minutes (must be > 0)
</ParamField>

<ParamField body="max_events_per_day" type="integer">
  Maximum bookable events per day (must be > 0)
</ParamField>

<ParamField body="min_advance_hours" type="integer">
  Minimum advance notice required in hours (must be >= 0)
</ParamField>

<ParamField body="business_hours" type="object">
  Weekly schedule configuration
  
  <ParamField body="monday" type="object">
    <ParamField body="enabled" type="boolean">
      Whether appointments are available
    </ParamField>
    <ParamField body="start" type="string">
      Opening time in HH:MM format (24-hour)
    </ParamField>
    <ParamField body="end" type="string">
      Closing time in HH:MM format (must be after start time)
    </ParamField>
  </ParamField>
  
  <ParamField body="tuesday" type="object">
    Same structure as Monday
  </ParamField>
  
  <ParamField body="wednesday" type="object">
    Same structure as Monday
  </ParamField>
  
  <ParamField body="thursday" type="object">
    Same structure as Monday
  </ParamField>
  
  <ParamField body="friday" type="object">
    Same structure as Monday
  </ParamField>
  
  <ParamField body="saturday" type="object">
    Same structure as Monday
  </ParamField>
  
  <ParamField body="sunday" type="object">
    Same structure as Monday
  </ParamField>
</ParamField>

### Validation Rules

<Check>
  **Duration**: Must be greater than 0 minutes
</Check>

<Check>
  **Max Events**: Must be greater than 0
</Check>

<Check>
  **Min Advance**: Must be 0 or positive
</Check>

<Check>
  **Time Format**: Must be HH:MM (24-hour format)
</Check>

<Check>
  **Time Range**: End time must be after start time
</Check>

<Check>
  **Business Days**: At least one day must be enabled
</Check>

<Check>
  **Timezone**: Must be a valid PHP timezone identifier
</Check>

### Response

<ResponseField name="success" type="boolean">
  Indicates if settings were saved successfully
</ResponseField>

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

<ResponseField name="error" type="string">
  Validation or system error message (only present on failure)
</ResponseField>

### Success Response Example

```json
{

  "message": "Configuración guardada exitosamente"
}
```

## Error Handling

| Status Code | Description |
|-------------|-------------|
| 200 | Settings saved successfully |
| 400 | Validation error (check error message for details) |
| 405 | Method not allowed (POST required for save) |
| 500 | Internal server error |

## Implementation Details

The save endpoint:
1. Validates all numeric constraints
2. Validates time formats using `App\Helpers\TimeValidator`
3. Ensures at least one business day is enabled
4. Uses database transactions for atomicity
5. Stores business hours as JSON per day
6. Logs changes for audit purposes

Source:
- GET: `api/get-calendar-settings.php:10-62`
- POST: `api/save-calendar-settings.php:28-145`

## Notes

<Info>
  These settings only take effect when calendar integration is enabled in the main bot settings (`calendarEnabled: true`).
</Info>

<Warning>
  Changing business hours does not affect already-booked appointments. Only new bookings will respect the updated schedule.
</Warning>
