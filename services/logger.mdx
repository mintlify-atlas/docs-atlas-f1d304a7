## Overview

The `Logger` class provides a lightweight file-based logging system for the WhatsApp RAG Bot. It supports multiple log levels and automatically organizes logs by date in the `logs/` directory.

<Note>
Logs are stored as plain text files with one entry per line, making them easy to parse and analyze with standard Unix tools like `grep`, `tail`, and `awk`.
</Note>

## Initialization

```php
use App\Core\Logger;

// Initialize with default path (logs/ directory)
$logger = new Logger();

// Or specify a custom log directory
$logger = new Logger('/var/log/whatsapp-bot');
```

The Logger automatically creates the log directory if it doesn't exist with `0755` permissions.

## Log Levels

The Logger supports four standard log levels:

| Level | Method | Use Case |
|-------|--------|----------|
| **INFO** | `info()` | General application events, successful operations |
| **WARNING** | `warning()` | Warning conditions, non-critical issues |
| **ERROR** | `error()` | Error conditions, failed operations |
| **DEBUG** | `debug()` | Detailed debugging information |

## Contextual Logging

Add structured context to log entries for better debugging:

```php
use App\Core\Logger;

$logger = new Logger();

// Log with context array
$logger->info('Message received', [
    'from' => '+1234567890',
    'message_id' => 'wamid.HBgLM...',
    'type' => 'text',
    'timestamp' => time()
]);

// Context is JSON-encoded
$logger->error('API request failed', [
    'endpoint' => 'https://graph.facebook.com/v17.0/messages',
    'status' => 401,
    'response' => 'Invalid access token'
]);
```

## Log File Structure

### File Organization

Logs are stored in daily files with the format `YYYY-MM-DD.log`:

```
logs/
├── 2026-03-04.log
├── 2026-03-05.log
└── 2026-03-06.log
```

### Log Entry Format

Each log entry follows this format:

```
[YYYY-MM-DD HH:MM:SS] [LEVEL] Message {"context":"json"}
```

**Example log file** (`logs/2026-03-06.log`):

```
[2026-03-06 10:15:23] [INFO] WhatsApp webhook received {"type":"message","phone":"+1234567890"}
[2026-03-06 10:15:24] [INFO] Processing text message {"length":42,"conversation_id":156}
[2026-03-06 10:15:25] [DEBUG] RAG search initiated {"query":"product pricing","chunks":5}
[2026-03-06 10:15:26] [INFO] AI response generated {"confidence":0.87,"tokens":234}
[2026-03-06 10:15:27] [INFO] Message sent to user {"message_id":"wamid.HBgL..."}
[2026-03-06 10:20:15] [ERROR] OpenAI API error {"status_code":429,"error":"Rate limit exceeded"}
[2026-03-06 10:20:16] [WARNING] Falling back to cached response
```

## Real-World Examples

<CodeGroup>
```php WhatsApp Message Processing
use App\Core\Logger;
use App\Services\WhatsAppService;

$logger = new Logger();

try {
    $logger->info('Webhook received', [
        'from' => $phoneNumber,
        'type' => $messageType
    ]);
    
    // Process message
    $response = $whatsapp->sendMessage($phoneNumber, $reply);
    
    $logger->info('Message sent successfully', [
        'to' => $phoneNumber,
        'message_id' => $response['messages'][0]['id']
    ]);
    
} catch (Exception $e) {
    $logger->error('Failed to send message', [
        'error' => $e->getMessage(),
        'phone' => $phoneNumber
    ]);
}
```

```php RAG Query Processing
use App\Core\Logger;
use App\Services\RAGService;

$logger = new Logger();
$rag = new RAGService();

$logger->debug('Starting RAG query', [
    'query' => $userMessage,
    'conversation_id' => $conversationId
]);

$results = $rag->search($userMessage, 5);

$logger->info('RAG search completed', [
    'results_count' => count($results),
    'top_score' => $results[0]['score'] ?? 0
]);

if (empty($results) || $results[0]['score'] < 0.7) {
    $logger->warning('Low confidence RAG results', [
        'score' => $results[0]['score'] ?? 0,
        'threshold' => 0.7
    ]);
}
```

```php Database Operations
use App\Core\Logger;
use App\Core\Database;

$logger = new Logger();
$db = Database::getInstance();

$logger->debug('Inserting document', [
    'filename' => $filename,
    'size' => $fileSize
]);

try {
    $db->beginTransaction();
    
    $docId = $db->insert('documents', $documentData);
    $logger->info('Document inserted', ['id' => $docId]);
    
    foreach ($chunks as $index => $chunk) {
        $db->insert('vectors', [
            'document_id' => $docId,
            'chunk_index' => $index,
            'chunk_text' => $chunk
        ]);
    }
    
    $db->commit();
    $logger->info('Document processing completed', [
        'document_id' => $docId,
        'chunks' => count($chunks)
    ]);
    
} catch (Exception $e) {
    $db->rollback();
    $logger->error('Document processing failed', [
        'error' => $e->getMessage(),
        'file' => $filename
    ]);
}
```

```php API Error Handling
use App\Core\Logger;
use App\Services\OpenAIService;

$logger = new Logger();
$openai = new OpenAIService();

try {
    $logger->debug('OpenAI API request', [
        'model' => 'gpt-3.5-turbo',
        'messages' => count($messages)
    ]);
    
    $response = $openai->chat($messages);
    
    $logger->info('OpenAI response received', [
        'tokens' => $response['usage']['total_tokens'],
        'finish_reason' => $response['choices'][0]['finish_reason']
    ]);
    
} catch (Exception $e) {
    $logger->error('OpenAI API error', [
        'error' => $e->getMessage(),
        'code' => $e->getCode()
    ]);
    
    // Log to settings for admin dashboard
    $db->update('settings', 
        ['setting_value' => $e->getMessage()],
        'setting_key = "openai_last_error"'
    );
}
```
</CodeGroup>

## Log Analysis

Use standard Unix tools to analyze logs:

<CodeGroup>
```bash View Recent Logs
# View last 50 log entries
tail -n 50 logs/2026-03-06.log

# Follow logs in real-time
tail -f logs/2026-03-06.log
```

```bash Filter by Level
# Show only errors
grep '\[ERROR\]' logs/2026-03-06.log

# Show errors and warnings
grep -E '\[(ERROR|WARNING)\]' logs/2026-03-06.log
```

```bash Search by Content
# Find all WhatsApp-related logs
grep 'WhatsApp' logs/2026-03-06.log

# Find logs for specific phone number
grep '+1234567890' logs/2026-03-06.log
```

```bash Count Errors
# Count errors today
grep -c '\[ERROR\]' logs/2026-03-06.log

# Count errors this week
grep -c '\[ERROR\]' logs/2026-03-*.log
```
</CodeGroup>

## Best Practices

<Card title="Use Appropriate Levels" icon="layer-group">
Use INFO for normal operations, WARNING for potential issues, ERROR for failures, and DEBUG for development.
</Card>

<Card title="Add Context" icon="circle-info">
Always include relevant context (IDs, timestamps, values) to make debugging easier.
</Card>

<Card title="Log Rotation" icon="rotate">
Implement log rotation to prevent disk space issues. Consider using `logrotate` on Linux.
</Card>

<Card title="Sensitive Data" icon="eye-slash">
Never log sensitive data like passwords, API keys, or full credit card numbers.
</Card>

## Log Rotation

On Linux systems, use `logrotate` to manage log files:

```bash /etc/logrotate.d/whatsapp-bot
/path/to/whatsapp-rag-bot/logs/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    create 0644 www-data www-data
}
```
